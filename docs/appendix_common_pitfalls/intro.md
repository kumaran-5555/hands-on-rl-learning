# 附录 A：强化学习训练调试指南

> **本附录目标**：强化学习训练过程的调试难度远高于监督学习——Reward 突然归零、Loss 震荡不收敛、GPU 显存溢出等问题，其报错信息往往指向错误的方向。本附录提供一套系统化的诊断框架，每种故障遵循"复现场景 → 理论分析 → 修复方案"三步流程。

## 为什么需要这份调试指南？

回顾第 1 章的 CartPole 和第 6 章的 PPO，你会发现训练曲线很少是一帆风顺的。但课程里用的参数都是调好的，问题被精心屏蔽了。一旦你离开课堂，面对自己的项目和真实环境，这些坑就会一个接一个地冒出来。

这份附录覆盖了四类最常见的训练故障，归为两个专题文件：

| 文件                                                   | 覆盖的坑                                                | 核心问题                         |
| ------------------------------------------------------ | ------------------------------------------------------- | -------------------------------- |
| [策略崩溃与奖励黑客](./policy-collapse-reward-hacking) | 策略崩溃（Policy Collapse）+ 奖励黑客（Reward Hacking） | 训练"跑飞了"怎么办？             |
| [显存爆炸与不收敛](./oom-nonconvergence)               | 显存爆炸（OOM）+ 训练不收敛（Non-convergence）          | 训练"跑不动"和"跑不动了"怎么办？ |

每个文件都包含可复现的代码、故障诊断图、以及经过验证的修复方案。建议你先通读一遍建立全局认识，然后在遇到具体问题时回来查阅对应章节。
