# 附录 B：强化学习工程实践指南

> 学会了 RL 算法之后，你会发现**真正的挑战不在算法，而在工程**。模型装不进一张卡、训练跑了一整天 loss 还在涨、评测分数和预期完全不符——这些问题在任何一本 RL 教材里都找不到答案，但在实际工作中你会天天遇到。
>
> 本附录每节只说清楚一件事。按需跳转即可。

## 本附录结构

| 节                                                  | 主题                                | 它解决什么问题                                                     |
| --------------------------------------------------- | ----------------------------------- | ------------------------------------------------------------------ |
| [B.1 RL 采样基础设施](./rl-infrastructure)          | RL 采样为什么是瓶颈，各规模怎么解决 | 从单机到 LLM 集群，采样瓶颈的变化和解法                            |
| [B.2 异步训练架构](./async-training)                | 生成和训练怎么并行跑                | 三种部署模式、权重同步、数据过期                                   |
| [B.3 分布式并行策略](./parallelism)                 | 模型太大一张卡装不下，怎么切        | DP/TP/PP/EP、混合精度、RL 特有挑战                                 |
| [B.4 Agentic RL 基础设施](./agentic-rl-infra)       | Agentic RL 需要什么基础设施         | 沙箱、轨迹存储、工具执行、信用分配                                 |
| [B.5 评测体系与 Badcase 分析](./evaluation-badcase) | 怎么知道模型好不好，不好怎么改      | Benchmark 选择、Badcase 归因、自动化闭环                           |
| [B.6 训练监控与故障排查](./monitoring)              | 训练出了问题怎么第一时间发现        | 监控指标、问题速查表                                               |
| [B.7 工业实战练习](./industrial-exercises)          | 综合练习                            | 5 道工业级练习，串联算法和工程知识                                 |
| [B.8 大模型 RL 训练指标词典](./metrics-glossary)    | 训练日志里的指标是什么意思          | PPO/GRPO/DPO/RM 训练指标按功能分组，含正常范围、异常信号和框架差异 |

## 阅读建议

- **如果你在做 LLM 后训练**：按 B.1 → B.2 → B.3 → B.5 → B.6 顺序阅读
- **如果你在做游戏/机器人 RL**：重点看 B.1 和 B.6
- **如果你在准备面试**：直接看 B.7 的练习，遇到盲区再往回翻
- **如果你在查指标含义**：直接看 [B.8 指标词典](./metrics-glossary)，按需查阅
