# 第 5 章：策略梯度与 Actor-Critic

上一章我们走了 Value-Based 路线——先算每个动作值多少分，再选分数最高的。DQN 在 CartPole 和 Atari 上效果很好，但它有一个根本性的局限：**只能处理有限个动作**。

CartPole 只有"左推"和"右推"两个选项，DQN 给每个动作算一个 Q 值，取最大的就行：

```
DQN 在 CartPole 上的做法

  状态: 小车位置、速度、杆子角度、角速度
         │
         ▼
  ┌─────────────────┐
  │   Q 网络         │
  │                 │
  │  Q(左推) = 0.3  │
  │  Q(右推) = 0.8  │──→ argmax → 选右推 ✓
  │                 │
  └─────────────────┘

  只有 2 个动作 → 挨个算一遍，轻松搞定
```

但如果要控制一个机械臂呢？肩关节、肘关节、腕关节各有多个自由度，每个自由度能施加 0 到 10 N·m 之间的任意力矩：

```
DQN 在机械臂上"傻眼"了

  状态: 6 个关节角度 + 6 个角速度
         │
         ▼
  ┌──────────────────────────────────────────┐
  │  力矩 = [0.0, 0.0, ..., 0.0] → Q = ?    │
  │  力矩 = [0.1, 0.0, ..., 0.0] → Q = ?    │
  │  力矩 = [0.0, 0.1, ..., 0.0] → Q = ?    │
  │  力矩 = [0.1, 0.1, ..., 0.0] → Q = ?    │
  │  ...                                     │
  │  力矩 = [9.9, 9.9, ..., 9.9] → Q = ?    │
  │  力矩 = [10.0, 10.0, ..., 10.0] → Q = ? │
  │                                          │
  │  连续值 × 6 个关节 = 无穷多种组合 ❌      │
  └──────────────────────────────────────────┘

  不可能给每种组合都算一个 Q 值！
```

更别说大模型生成文本的场景了——每一步要从几万个 token 里选，策略本身是一个连续的概率分布，DQN 那套"挨个算分数再取最大"的思路完全行不通。

## 换一条路：直接学策略

既然"先打分再选"走不通，那就换一条路：**跳过 Q 值，直接学策略**。不问"每个动作值多少分"，而是直接学"在什么情况下该做什么"。

就像你学骑自行车——你不需要先算出"左转 15 度值 0.87 分"，而是直接练出"感觉这样骑更稳"的直觉。这条路线叫做 **Policy-Based 方法**，策略梯度（Policy Gradient）就是这条路线的理论核心。

```
两条路线的对比

  Value-Based（第 4 章 DQN）           Policy-Based（本章）
  ────────────────────────           ──────────────────────

  状态 s                                状态 s
    │                                     │
    ▼                                     ▼
  ┌─────────┐                          ┌───────────┐
  │ Q 网络   │                          │ 策略网络   │
  │         │                          │           │
  │ Q(s,a₁) │                          │ π(a₁|s)=0.2│
  │ Q(s,a₂) │──→ argmax ──→ a*        │ π(a₂|s)=0.7│──→ 按概率采样 → a
  │ Q(s,a₃) │   (选分数最高的)         │ π(a₃|s)=0.1│   (直接输出概率)
  └─────────┘                          └───────────┘

  先打分，再选                         直接输出"该做什么"
  连续空间算不出 argmax                 天然支持连续动作 ✓
```

## 这一章怎么展开？

我们从最简单的一个赌博游戏开始，一步步把策略梯度讲清楚。整章有一条贯穿始终的暗线——**策略梯度最大的敌人是"噪声太大"**，每一节的改进都在想办法压制噪声：

```
本章路线图（暗线：一直在降噪声）

  5.1 摇骰子         5.2 策略梯度定理       5.3 Actor-Critic       5.4 基线实验
  ──────────        ──────────────        ───────────────        ──────────────

  让 AI 玩赌博机     为什么那一行代码       怎么治噪声？           亲眼看差距
  (A:30% B:70%)      能工作？

  AI 能学会选 B      数学证明：             关键洞察：             有基线 vs 无基线
  但过程很抖          "好结果 → 强化        不要问"拿了多少分"     的训练曲线对比
                      该动作的概率"         要问"比平时好了多少"
       │                   │                     │                    │
       ▼                   ▼                     ▼                    ▼

  看到噪声的         理解噪声的             引入 Critic 网络       收敛更快更稳
  具体表现           数学来源               噪声大幅降低
                           │                      │
                           └──── 暗线：降噪声 ────┘
```

1. **先动手**：让一个 AI 玩两臂赌博机（A 赢率 30%，B 赢率 70%），看它能不能自己发现"永远选赢率高的那个"。你会发现它确实能学会，但学得很抖——有时候明明选对了，偏偏那次运气差输了，它就被误导了。这就是噪声太大的直接表现。

2. **再讲原理**：解释为什么 `loss = -log_prob * reward` 这一行代码能让 AI 学会"偏爱好的动作"。背后是一个叫"策略梯度定理"的数学结论。理解了它，你就理解了所有策略梯度算法的共同骨架。

3. **然后解决噪声问题**：关键洞察是——不要问"这趟跑了多少分"，而要问"这趟比平时好了多少"。减掉一个"基准线"，同样的数学方向，噪声却小了很多。而最好的基准线恰好就是第 3 章学的价值函数 $V(s)$——它天然引出了第二个网络（Critic），和原来的策略网络（Actor）组成了 **Actor-Critic 架构**。

4. **最后用实验验证**：回到赌博机环境，跑有基线和无基线的对比实验，亲眼看到差距。

这个 Actor-Critic 的双网络结构，是后续 PPO、DPO、GRPO 的共同骨架——不理解它，就看不懂大模型对齐的任何一个算法。
